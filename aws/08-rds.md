# Day22 RDS（MySQL）

## 環境
・AWS  
・EC2（Ubuntu）  
・RDS（MySQL / db.t3.micro）  

## 概要
RDS（MySQL）インスタンスを作成し、
EC2から接続できる構成を構築する。

セキュリティグループによるアクセス制御と、
マネージドデータベースの基本的な利用方法を理解する。

---

## 作業手順

### 1. RDS作成
1. RDS → データベース作成
2. エンジン：MySQL
3. テンプレート：無料利用枠
4. DBインスタンス識別子：my-rds
5. マスターユーザー：admin
6. インスタンスサイズ：db.t3.micro

---

### 2. 接続設定
・VPC：EC2と同じ  
・パブリックアクセス：有効  

---

### 3. セキュリティグループ設定
RDSのセキュリティグループに以下を追加

・タイプ：MYSQL/Aurora  
・ポート：3306  
・ソース：EC2のセキュリティグループ  

---

### 4. MySQLクライアントインストール
```
sudo apt update
sudo apt install mysql-client -y
```

---

### 5. RDSへ接続
```
mysql -h RDSエンドポイント -u admin -p
```

---

### 6. 動作確認
```
show databases;
```

---

## 確認
・EC2からRDSへ接続できること  
・データベース一覧が取得できること  

---

## 補足
・RDSはマネージドなリレーショナルデータベース  
・バックアップやパッチ適用などをAWSが管理する  
・セキュリティグループで接続元を制御する  
・IPではなくセキュリティグループで制御するのがベストプラクティス  
・実務ではプライベートサブネットに配置することが多い  

---

## 学び
・RDSは運用を簡略化できるデータベースサービスである  
・セキュリティグループによるアクセス制御が重要である  
・EC2を経由した構成にすることでセキュリティを高められる  
・クラウドではネットワーク設計とアクセス制御が密接に関係していると理解した
