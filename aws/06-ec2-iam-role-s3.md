# Day20 EC2からS3へアクセス（IAMロール）

## 環境
・AWS
・EC2（Ubuntu）
・S3（Day19で作成済みバケット）
・IAM
・AWS CLI

## 概要
EC2インスタンスにIAMロールを付与し、
アクセスキーを使用せずにS3へアクセスする構成を構築する。

既存のS3バケット（Day19で作成）を利用し、
実務に近いリソース連携と権限設計を理解する。

## 作業手順

### 1. IAMロール作成
1. IAM → ロール → 作成
2. 信頼されたエンティティタイプ：AWSサービス
3. ユースケース：EC2
4. 許可ポリシー：
   ・AmazonS3ReadOnlyAccess
5. ロール名：ec2-s3-role

---

### 2. EC2へロールをアタッチ
1. EC2 → インスタンス
2. 対象インスタンスを選択
3. アクション → セキュリティ → IAMロールを変更
4. 作成したロールを選択

---

### 3. EC2へSSH接続
```
ssh -i キーペア.pem ubuntu@パブリックIP
```

---

### 4. AWS CLIインストール（未導入の場合）
Ubuntu環境でawsコマンドが存在しなかったためインストールを実施

```
sudo apt update
sudo apt install awscli -y
sudo apt upgrade awscli -y
```

---

### 5. S3バケット一覧取得
```
aws s3 ls
```

---

### 6. バケット内のオブジェクト一覧確認
```
aws s3 ls s3://バケット名
```

---

### 7. ファイル内容の確認（ダウンロードして確認）
```
aws s3 cp s3://バケット名/ファイル名 .
cat ファイル名
```

---

### 8. ファイル内容の直接確認（標準出力）
```
aws s3 cp s3://バケット名/ファイル名 -
```

---

## 確認
・EC2からS3のバケット一覧が取得できること  
・Day19で作成したバケット内のオブジェクト一覧が確認できること  
・ファイルをダウンロードして内容が確認できること  
・標準出力で直接内容が確認できること  
・アクセスキー未設定（aws configure未実施）でも実行できること  

---

## 学び
・EC2からAWSサービスへ安全にアクセスするにはIAMロールを使用する必要がある  
・アクセスキーを使わない設計がセキュリティ上重要である  
・S3はオブジェクトストレージであり、ファイルの扱い方がローカルと異なることを理解した  
・ファイル内容を確認するにはダウンロードまたは標準出力を利用する必要がある  
・環境（Amazon Linux / Ubuntu）による差異を考慮する重要性を学んだ  
・IAMは「誰がどのリソースに何をできるか」を定義する設計であると理解した
